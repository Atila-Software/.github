# Security Policy

Atila Software takes security reports seriously. If you believe you have found a vulnerability in one of our repositories, please report it privately.

## Reporting a Vulnerability

Do not open a public GitHub issue for sensitive security problems.

Please contact us through one of the following channels:

- Security email: TODO
- Internal security contact: TODO
- Alternative private channel: TODO

Include as much detail as possible:

- Repository or project affected
- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Any suggested mitigation, if known

## Response

We will review the report, assess the impact, and coordinate a fix when needed. Response times may vary depending on severity and project ownership.

## Responsible Disclosure

Please avoid accessing, modifying, or sharing data that does not belong to you. Give us reasonable time to investigate and address the issue before public disclosure.
