# Atila Software

We build software that helps businesses move faster, work cleaner, and make better decisions.

Atila Software is a software company from Bell Ville, Cordoba, Argentina, focused on business management platforms, vertical market solutions, mobile apps, integrations, support workflows, and custom software development.

Our job is simple to say and hard to do well: turn messy operational work into clear, useful, reliable systems.

## What We Build

Our ecosystem includes products and services for companies that need practical tools, real support, and software that understands daily operations.

- **Atila Online**: business administration for Argentina and Chile, including purchases, sales, stock, electronic invoicing, accounting, mobile access, and Power BI dashboards.
- **Atila Agro**: management for agricultural businesses, including costs, fields, lots, tasks, operators, commercialization, administration, and accounting.
- **Atila Turismo**: tools for wholesale, retail, and student travel operations.
- **Atila Postventa**: service, support, claims, technical service, task tracking, and customer follow-up.
- **Atila IA**: intelligent workflows for supplier invoices, company data queries, decision support, and image analysis.
- **Atila Market**: wholesale order management with sales, stock, administration, accounting, mobile access, and dashboards.
- **Software Factory**: custom digital solutions for business needs that do not fit neatly into a box.

## Core Technologies

We use technology as a toolbox, not a religion. The stack depends on the problem, but these are common around here:

- **Flutter and Dart** for mobile and cross-platform apps
- **.NET** for APIs, backend services, and enterprise systems
- **Node.js and JavaScript** for services, tooling, and integrations
- **SQL Server** for business data, reporting, and operational logic
- **Docker** for reproducible environments and deployment workflows
- **Power BI** for dashboards and business visibility
- **OpenAI-powered workflows** where automation can remove repetitive work and surface useful insights

## How We Work

We like software that earns its keep.

- Small, reviewable pull requests.
- Clear issue descriptions and reproducible bugs.
- Practical documentation that helps the next person move faster.
- Stable main branches and deliberate releases.
- Strong support loops with real user feedback.
- Simple interfaces for complex business rules.
- Quality, maintainability, and security treated as everyday engineering work.

We prefer boring reliability over clever surprises, but we are absolutely in favor of delightful details when they make work easier.

## For New Contributors

Welcome. The fastest way to get productive is to read the repo before changing it.

Start with:

- The repository `README.md`
- `CONTRIBUTING.md`
- The pull request template
- Project-specific setup, architecture, or deployment docs
- Existing issues and recent pull requests for context

Good contributions are not just code. A clear bug report, a better setup note, a reproducible test case, or a screenshot that saves someone ten minutes also counts.
